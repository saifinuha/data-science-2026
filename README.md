## 🚀 Cara Menjalankan Notebook

### **Opsi 1: Google Colab (Direkomendasikan)**

Semua notebook telah dioptimalkan untuk berjalan di Google Colab. Langkah-langkahnya:

1. Buka notebook dari link di tabel di atas
2. Klik tombol **"Open in Colab"** yang ada di bagian atas notebook
3. Jalankan setiap cell dengan menekan `Shift + Enter` atau klik tombol ▶️ di sebelah cell
4. Jika ada dataset dari URL, Colab akan otomatis mengunduhnya

**Keuntungan Google Colab:**
- ✅ Gratis dan tidak perlu instalasi
- ✅ GPU tersedia jika diperlukan
- ✅ Semua library sudah pre-installed

### **Opsi 2: Jupyter Notebook Lokal**

Jika Anda ingin menjalankan di komputer lokal:

#### **Prasyarat:**
```bash
# Install Python 3.x dan pip
python --version
pip --version
```

#### **Langkah Instalasi:**

1. **Clone atau download repository:**
   ```bash
   git clone https://github.com/saifinuha/data-science-2026.git
   cd data-science-2026
   ```

2. **Buat virtual environment (opsional namun direkomendasikan):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Untuk Linux/Mac
   # atau
   venv\Scripts\activate  # Untuk Windows
   ```

3. **Install dependencies:**
   ```bash
   pip install jupyter pandas numpy matplotlib seaborn scikit-learn scipy
   ```

4. **Jalankan Jupyter Notebook:**
   ```bash
   jupyter notebook
   ```

5. **Buka file notebook** (misalnya `Pertemuan1_Saifin_Nuha_240401010257.ipynb`) dari browser yang terbuka, kemudian jalankan setiap cell.

## 📊 Kesimpulan Perjalanan Belajar Data Science (Pertemuan 1–7)

### **Pertemuan 1: Fondasi Python**
Memulai dengan dasar-dasar Python: variabel, tipe data, list, dictionary, dan fungsi. Ini adalah fondasi penting untuk memahami konsep-konsep Data Science selanjutnya.

### **Pertemuan 2: NumPy, Pandas & Eksplorasi Data**
Belajar menggunakan NumPy untuk operasi numerik array dan Pandas untuk manipulasi DataFrame. Melakukan EDA pertama pada dataset Titanic untuk memahami struktur data, missing values, dan statistik deskriptif.

**Key Insight:** ~38.4% penumpang Titanic selamat, dengan tingkat keselamatan tertinggi pada kelas 1 (63%) dan terendah pada kelas 3 (24.2%).

### **Pertemuan 3: Data Cleaning**
Mempelajari pipeline pembersihan data yang sistematis: menghapus duplikat, normalisasi string, imputasi missing values, dan penanganan outlier menggunakan metode IQR. Data yang bersih adalah fondasi untuk analisis yang akurat.

### **Pertemuan 4: Analisis Deskriptif & Visualisasi**
Menggunakan Matplotlib dan Seaborn untuk visualisasi data (histogram, boxplot, violin plot, scatter plot, heatmap). Menganalisis dataset Iris untuk menemukan pola: korelasi tertinggi antara petal_length dan petal_width (0.963).

### **Pertemuan 5: Dashboard & EDA Komprehensif**
Membuat dashboard multi-panel untuk analisis Titanic yang lebih dalam. Visualisasi hubungan antar variabel melalui berbagai sudut pandang, menemukan pola kelangsungan hidup berdasarkan kelas, usia, dan gender dengan pendekatan **"What, So What, Now What"**.

### **Pertemuan 6: Feature Engineering & Preprocessing**
Mempelajari langkah-langkah penting sebelum ML: one-hot encoding untuk kategorikal, train-test split dengan stratifikasi, dan feature scaling dengan StandardScaler. Semua teknik ini memastikan data siap untuk modeling.

### **Pertemuan 7: Machine Learning & Model Evaluation**
Membangun model regresi linear untuk memprediksi gaji berdasarkan pengalaman, pendidikan, dan kota. Mengevaluasi performa model dengan metrik: MAE, RMSE, R², dan analisis residual.

**Key Result:** Model mencapai R² = 0.974 (97.4% varians dijelaskan), menunjukkan pengalaman adalah faktor terkuat dalam menentukan gaji.

---

### **Pembelajaran Utama dari Keseluruhan Perjalanan:**

1. **Data Science adalah Iteratif:** Dari eksplorasi → cleaning → visualisasi → modeling adalah satu alur yang saling terkait
2. **Data Quality Matters:** 80% pekerjaan adalah data cleaning; data yang baik menghasilkan insight yang baik
3. **Visualisasi adalah Komunikasi:** Grafik yang tepat dapat mengungkap pola yang tersembunyi dalam angka
4. **Domain Understanding Penting:** Memahami konteks bisnis membantu memilih fitur dan metrik evaluasi yang tepat
5. **Reproducibility:** Menggunakan `random_state` dan dokumentasi kode memastikan hasil yang konsisten
